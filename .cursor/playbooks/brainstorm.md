# Playbook: brainstorm (ex-`/brainstorm`)

## Objetivo

Explorar **opcoes e trade-offs** antes de implementar; **sem codigo** nesta fase.

## Entrada

- Tema ou decisao (ex.: autenticacao, modelo de dados, estrategia de cache).

## Passos

1. Contextualizar problema, usuario e restricoes.
2. Gerar **no minimo tres** abordagens distintas quando fizer sentido.
3. Para cada opcao: beneficios, custos, esforco (baixo/medio/alto), riscos.
4. Recomendar uma opcao com justificativa; deixar decisao final com o usuario.

## Formato sugerido

- Contexto
- Opcao A / B / C (pros, contras, esforco)
- Recomendacao e perguntas em aberto

## Regras

- Nao escrever codigo de producao aqui.
- Preferir honestidade sobre complexidade a “vender” uma unica solucao.

## Saida esperada

- Documento ou mensagem estruturada pronta para alimentar o playbook `plan.md` ou implementacao.
