# Architecture

## Principio central

Simplicidade primeiro; complexidade somente quando justificada por requisito real.

## Fluxo de decisao

1. Entenda requisitos e restricoes.
2. Liste opcoes viaveis.
3. Compare trade-offs (custo, risco, manutencao, performance).
4. Escolha a opcao mais simples que atende ao contexto.
5. Registre a decisao quando houver impacto arquitetural.

## Checklist

- Requisitos estao claros.
- Alternativas foram consideradas.
- Trade-offs foram explicitados.
- Impacto em componentes dependentes foi analisado.
