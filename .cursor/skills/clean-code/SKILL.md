# Clean Code

## Regras

- Resolva o pedido com a menor complexidade possivel.
- Evite over-engineering, helpers desnecessarios e abstracoes prematuras.
- Prefira nomes claros e intencionais para funcoes e variaveis.
- Use guard clauses para reduzir aninhamento.
- Mantenha funcoes pequenas e com responsabilidade unica.
- Evite comentarios obvios; prefira codigo autoexplicativo.

## Checklist rapido

- O codigo atende exatamente ao pedido?
- Ha impacto em arquivos dependentes?
- A mudanca pode ser validada de forma objetiva?
